# Carbon Typography

**Source:** https://carbondesignsystem.com/elements/typography/overview/

---

## IBM Plex Typeface

Carbon uses **IBM Plex** — IBM's open-source corporate typeface, designed
in-house to reflect IBM's precision, clarity, and human quality.

| Variant | Usage |
|---|---|
| **IBM Plex Sans** | Primary — UI, body, headings |
| **IBM Plex Serif** | Editorial — long-form reading, marketing |
| **IBM Plex Mono** | Code — technical content, terminal output |

All three are free at [fonts.google.com/featured/Plex](https://fonts.google.com/featured/Plex)
and via [IBM's GitHub](https://github.com/IBM/plex).

### Font Stacks

```css
font-family: 'IBM Plex Sans', 'Helvetica Neue', Arial, sans-serif;
font-family: 'IBM Plex Serif', 'Georgia', Times, serif;
font-family: 'IBM Plex Mono', 'Menlo', 'DejaVu Sans Mono', Courier, monospace;
```

## Weights

Carbon uses only 3 weights from IBM Plex:

| Weight | Value | Token Usage |
|---|---|---|
| **Light** | 300 | Large display headings, editorial |
| **Regular** | 400 | Body text, paragraphs, most headings |
| **SemiBold** | 600 | UI headings, buttons, emphasized text |

> Bold (700) is **not used** in Carbon. If you need stronger emphasis than SemiBold, reconsider the design.

## Type Scale

Carbon's scale is generated by a mathematical formula from a 12px base.

$$\text{Type scale formula:} \quad X_n = X_{n-1} + \left(\left\lfloor\frac{n-2}{4}\right\rfloor + 1\right) \times 2$$

| Step | Size | rem | Notes |
|---|---|---|---|
| 1 | 12px | 0.75rem | Code, labels, captions |
| 2 | 14px | 0.875rem | Body compact, helper text |
| 3 | 16px | 1.0rem | Body, standard UI text |
| 4 | 18px | 1.125rem | — |
| 5 | 20px | 1.25rem | Productive heading 03 |
| 6 | 24px | 1.5rem | — |
| 7 | 28px | 1.75rem | Productive heading 04 |
| 8 | 32px | 2.0rem | — |
| 9 | 36px | 2.25rem | Productive heading 05 |
| 10 | 42px | 2.625rem | Productive heading 06, Display 01 |
| 11 | 48px | 3.0rem | Expressive heading 06 |
| 12 | 54px | 3.375rem | Productive heading 07 |
| 13 | 60px | 3.75rem | — |
| 14 | 68px | 4.25rem | Large editorial |
| 15 | 76px | 4.75rem | — |
| 16 | 84px | 5.25rem | — |
| 17 | 92px | 5.75rem | Hero scale |

## Two Type Sets

Carbon defines two parallel type sets for different product contexts:

### Productive Type Set (Product / Application UI)

For IBM Cloud, Watson products, dashboards, and complex UIs.
Fixed sizes — do not respond to viewport width.

| Token | Size | Weight | Line Height | Usage |
|---|---|---|---|---|
| `$label-01` | 12px | 400 | 1.333 | Form labels, captions |
| `$helper-text-01` | 12px | 400 *italic* | 1.333 | Form helper text, hints |
| `$code-01` | 12px | 400 (mono) | 1.333 | Inline code |
| `$code-02` | 14px | 400 (mono) | 1.428 | Code blocks |
| `$body-compact-01` | 14px | 400 | 1.285 | Single-line body, rows |
| `$body-01` | 14px | 400 | 1.428 | Multi-line body text |
| `$body-compact-02` | 16px | 400 | 1.375 | Medium density paragraphs |
| `$body-02` | 16px | 400 | 1.5 | Long-form paragraph text |
| `$heading-01` | 14px | 600 | 1.285 | Component heading (small) |
| `$heading-02` | 16px | 600 | 1.375 | Component heading (medium) |
| `$heading-03` | 20px | 400 | 1.4 | Section heading |
| `$heading-04` | 28px | 400 | 1.285 | Page heading |
| `$heading-05` | 36px | 400 | 1.222 | Major heading |
| `$heading-06` | 42px | 300 (Light) | 1.238 | Hero heading |
| `$heading-07` | 54px | 300 (Light) | 1.111 | Display span |

### Expressive Type Set (Editorial / Marketing)

For IBM.com, marketing pages, and high-impact storytelling.
Key headings are **responsive** — size changes at breakpoints.

| Token | Base Size | Weight | Responsive? |
|---|---|---|---|
| `$expressive-heading-01` | 14px | 600 | No |
| `$expressive-heading-02` | 16px | 600 | No |
| `$expressive-heading-03` | 20px | 400 | No |
| `$expressive-heading-04` | 28px | 400 | No |
| `$expressive-heading-05` | 36px | 400 | **Yes** — scales up at lg/xlg/max |
| `$expressive-heading-06` | 48px | 300 (Light) | **Yes** — scales up at lg/xlg/max |
| `$display-01` | 42px | 300 (Light) | **Yes** |
| `$display-02` | 42px | 600 (SemiBold) | **Yes** |
| `$expressive-paragraph-01` | 24px | 300 | No |
| `$quotation-01` | 20px | 400 *italic* | No |
| `$quotation-02` | 32px | 300 *italic* | No |

## Type Color Rules

| Context | Token | Explanation |
|---|---|---|
| Body text | `$text-primary` | Neutral first choice |
| Secondary / supporting | `$text-secondary` | Supplemental information |
| Form helper | `$text-helper` | Non-critical guidance |
| Placeholder | `$text-placeholder` | Not real content — decorative |
| Error message | `$text-error` | Always `$support-error` or `$text-error` |
| Links (body copy) | `$link-primary` | IBM Blue 60; must be underlined in paragraphs |
| Links (standalone) | `$link-primary` | CTA links; underline optional if icon present |
| On colored surface | `$text-on-color` | White on button-primary, dark tokens |
| Code text | `$text-primary` + mono font | Never use colored text for code |

## Line Spacing

Carbon specifies line height as a **ratio of the type size**, not absolute px:

- **Tight (compact):** 1.25–1.285 — for UI labels, data tables, short spans
- **Standard:** 1.375–1.4 — for body copy in components
- **Loose:** 1.5 — for long-form paragraph reading
- **Display:** 1.1–1.25 — for large headings (line height decreases as size increases)

## Anti-Patterns

| Wrong | Right |
|---|---|
| Bold (700) weight | SemiBold (600) max |
| Three or more type sizes on one slide | Two max — heading + body |
| Italic for decorative → italic quotation | Use `$quotation-01` token only |
| Gray text below 4.5:1 contrast | Use `$text-secondary` (≥ 4.5:1) |
| All-caps for headings | Not a Carbon pattern — avoid |
| Serif for UI elements | IBM Plex Serif = editorial only |
